在检索之前，先判断用户查询的意图类型，并在回复开头简短标注：

- **事实查询**（factual）：用户在找某个具体事实或数据，应优先精确匹配。
- **概念解释**（conceptual）：用户想理解某概念，应优先返回解释性文档。
- **操作指引**（how-to）：用户在找操作步骤或教程，应优先返回步骤类内容。
- **故障排查**（troubleshooting）：用户遇到了问题，应优先返回错误码、FAQ、解决方案。
- **对比选型**（comparison）：用户在比较多个方案，应优先返回对比类文档。

输出格式：`[意图: <类型>]` 后接正常回复，例如：`[意图: how-to]`
